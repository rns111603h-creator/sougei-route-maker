# 送迎ルートメーカー

福祉事業所向けの、静的Webアプリです。事業所の住所と利用者の住所を入力すると、ブラウザ内で走行時間が短くなる巡回順を計算し、地図表示・道路距離と走行時間の目安・Googleマップ起動・印刷用ルート表の作成まで行えます。

バックエンド、データベース、ログイン、ビルド工程、npm依存はありません。`index.html` をそのまま開いて使えます。

## できること

- 事業所住所と送迎利用者の名前・通常地点・住所を入力できます。
- 最大8つのコースをタブで切り替え、コース名とコース連絡先を自由に変更できます。
- 送り地点が通常地点と違う利用者だけ、送り専用の場所名・住所を追加できます。
- コース名、連絡先、対象月、ルート計算日、利用者名、場所名、住所、送り地点、休み日をこの端末のブラウザ内に保存できます。
- コース設定を暗号化した `.sougei` 共有ファイルとして書き出し、別端末で読み込めます。
- 送迎利用者リストは1つのまま、「迎えルート / 送りルート」を選んでルート計算できます。
- 「最後に事業所へ戻る（周回ルート）」で往復・片道を切り替えられます。
- 場所名・施設名または住所を OpenStreetMap Nominatim で緯度経度へ変換します。
- 場所名と住所の両方がある場合は、場所名を先に検索し、見つからない時に住所を検索します。
- 番地まで見つからない住所は、丁目などの代表地点で検索を補助します。
- 複数住所は無料サービスへの配慮として約1秒間隔で順番に検索します。
- 巡回順はブラウザ内で、OSRMの道路時間を使った最近傍法 + 2-opt 改善により計算します。
- OSRM で高速道路を避けた道路時間の比較、道路距離、走行時間の目安を取得します。OSRMの公開サーバーが高速道路除外指定に対応しない場合は、通常の道路時間を目安として表示し、画面上にその旨を明記します。
- 出発時刻と停車時間から、各ポイントの到着予定時刻を自動で表示します。
- 到着予定表の時刻を手入力で調整できます。調整した地点以降の時刻は、区間時間と停車時間に合わせて自動で再計算されます。
- 利用者ごとに休み日を日付でチェックできます。月末の週では、次月の平日分も同じ週として選択できます。
- 休みチェックが入っている利用者もルート計算には含め、印刷用送迎表で休みの日が分かるように表示します。
- ルート計算後、送迎利用者リストは計算されたルート順へ並び替わります。
- 計算後のルート順は、結果リスト上でドラッグまたは上下ボタンにより手動変更できます。変更後は道路ルートと到着予定を再計算します。
- Leaflet + OpenStreetMap タイルで、発 → 1 → 2 ... のマーカーと道路ルート線を表示します。
- GoogleマップのURLスキームで、計算した順番のまま高速道路・有料道路を避ける設定のナビを開けます。
- コースごとに、1ヶ月分または選択中の週だけの送迎表をA4タテで印刷できます。1ページは平日5日分です。月末の週は次月の平日まで同じシートに表示し、休み予定は「休」、祝日は灰色で表示します。
- 現在地を事業所位置として、この端末のブラウザ内に保存できます。

## 使い方

1. `index.html` をブラウザで開きます。
2. 使うコースタブを選び、必要に応じてコース名、コース連絡先、送迎表の対象月、ルート計算日を変更します。
3. 事業所の住所を入力します。
4. 必要に応じて「最後に事業所へ戻る（周回ルート）」を切り替えます。
5. 送迎利用者リストに、氏名、場所名・施設名、住所を入力します。行は追加・削除できます。
6. 送りの乗降地点が違う利用者だけ、「送り地点が違う場合だけ入力」を開き、送りの場所名・住所を入力します。
7. 休み予定がある場合は、各利用者行の「休み日を選択」を開き、休みの日にチェックを入れます。
8. 到着予定を出したい場合は、出発時刻と1件あたり停車時間を入力します。
9. 「迎えルート / 送りルート」から今回計算するルートを選びます。
10. 「ルートを計算する」を押します。
11. 地図、時間優先の巡回順、道路距離、走行時間、到着予定を確認します。計算後、送迎利用者リストはルート順に並び替わります。
12. 順番を変えたい場合は、結果リストの利用者をドラッグするか、上下ボタンで移動します。移動後は道路ルートと到着予定が再計算されます。
13. 交通状況や現場判断で時刻を調整したい場合は、到着予定表の時刻欄を手入力します。以降の予定時刻も自動で更新されます。
14. 必要に応じて「コース設定を保存」を押し、コース名・連絡先・送迎利用者リスト・休み日をこの端末に保存します。
15. 必要に応じて「Googleマップで開く」または「ルート表を印刷」を使います。

事業所にいる状態で「現在地を事業所として保存」を押すと、その端末だけで事業所座標を保存できます。次回以降は「保存した事業所位置を使う」で、住所検索を省いて事業所の発着地として使えます。

「コース設定を保存」を押すと、8コース分のコース名、連絡先、対象月、ルート計算日、利用者名、通常の場所名・住所、送り専用の場所名・住所、休み日をこの端末のブラウザ内に保存します。保存した内容は、同じ端末・同じブラウザで次回開いた時に読み込まれます。不要になった場合は「保存内容を削除」を押してください。

「共有ファイルを書き出し」を押すと、8コース分の設定を暗号化した `.sougei` ファイルとして保存できます。ファイルには氏名、場所名、住所、送り地点、休み日などの個人情報が含まれるため、合言葉はファイルと別の方法で共有してください。「共有ファイルを読み込み」で別端末に取り込めます。読み込んだだけではその端末に保存されないため、継続して使う場合は読み込み後に「コース設定を保存」を押してください。共有ファイルには計算済みルート、座標、現在地保存情報は含めません。

ルート計算日は、印刷する週の基準日として使います。休みチェックが入っている利用者もルート計算には含めます。印刷用の送迎表では、1ヶ月分またはルート計算日を含む月曜〜金曜の5日分を選んで印刷できます。月末の週は次月の平日も同じシートに出します。休みチェックが入った日は「休」、祝日は灰色で表示します。通常日は「迎」「送」の2枠で、迎え・送りの利用状況をチェックできます。

立ち寄り先は、場所名・施設名が入力されていれば住所より先に検索します。場所名で見つからない場合に住所を検索するため、まず施設名や地図上の登録名を入れ、補助として住所を入れる運用ができます。送り地点が入力されている利用者は、送りルート計算時だけ送り地点を使います。印刷用ルート表には場所名・施設名のみを表示し、送り地点が違う人だけ「迎：... / 送：...」の形で両方を表示します。住所だけ入力して場所名が空欄の場合、印刷の場所欄は空欄になります。

住所が見つからない場合は、その行が赤く表示されます。市町村名を含める、施設名にする、または丁目までの住所に直して再計算してください。番地まで見つからない住所は、丁目などの代表地点として表示される場合があります。

## GitHub Pages での公開手順

1. このフォルダを Public リポジトリとして GitHub に push します。
2. GitHub のリポジトリ画面で `Settings` を開きます。
3. 左メニューの `Pages` を開きます。
4. `Build and deployment` の `Source` で `Deploy from a branch` を選びます。
5. `Branch` で `main` / `(root)` を選びます。
6. `Save` を押します。
7. 数分後に表示される GitHub Pages のURLへアクセスします。

GitHub Pages のプロジェクトサイトで壊れないよう、CSSとJavaScriptは `css/style.css`、`js/app.js` の相対パスで読み込んでいます。また、Jekyll 処理を避けるため `.nojekyll` を置いています。

## プライバシー注意

- 「コース設定を保存」を使った場合、コース名、連絡先、対象月、ルート計算日、利用者名、場所名、住所、送り地点、休み日がこの端末のブラウザ内に保存されます。
- 「共有ファイルを書き出し」を使った場合、コース名、連絡先、対象月、ルート計算日、利用者名、場所名、住所、送り地点、休み日が暗号化された `.sougei` ファイルに含まれます。合言葉は保存されません。
- 「現在地を事業所として保存」を使った場合、事業所の緯度経度がこの端末のブラウザ内に保存されます。
- データベース、ログイン、アクセス解析は使っていません。
- 氏名は外部サービスへ送信されません。
- 場所名・施設名・住所は緯度経度へ変換する時だけ OpenStreetMap Nominatim へ送信されます。
- 道路時間の比較、道路距離、走行時間を取得する時、事業所と立ち寄り先の座標が OSRM へ送信されます。OSRMには高速道路を避ける指定を付けています。
- 「Googleマップで開く」を押した場合、計算した順番の事業所・立ち寄り先の座標が Google マップへ渡されます。Googleマップには高速道路・有料道路を避ける指定を付けています。
- APIキー、トークン、シークレットは使っていません。

業務で使う場合は、送信される住所情報の扱いについて事業所内の個人情報ルールに沿って確認してください。

## 技術メモ

- HTML / CSS / JavaScript のみで構成しています。
- 地図は Leaflet と OpenStreetMap タイルを CDN から読み込みます。
- 住所検索は OpenStreetMap Nominatim の無料APIを利用します。
- 道路時間の比較は OSRM の Table API、道路ルートと走行時間は OSRM の Route API を利用します。まず `exclude=motorway` を付けて高速道路除外を試し、OSRMの公開サーバーが未対応の場合は通常の道路時間へフォールバックします。
- Googleマップは `https://www.google.com/maps/dir/?api=1...` のURLを生成し、`avoid=highways,tolls` を付けて開きます。
- 道路距離と走行時間は目安です。渋滞、信号待ち、乗降介助時間は反映されません。
