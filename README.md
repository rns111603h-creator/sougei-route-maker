# 送迎ルートメーカー

福祉事業所の送迎順をブラウザだけで作成する、完全静的なWebアプリです。バックエンド、データベース、ログイン、ビルド工程、npm依存はありません。GitHub Pages のプロジェクトサイトとして、リポジトリ直下の `index.html` をそのまま公開できます。

## 主な機能

- 事業所住所と利用者名・住所を入力して、送迎順を自動計算
- OpenStreetMap Nominatim による住所検索（無料API・APIキー不要）
- 無料サービスに配慮し、複数住所は約1秒間隔で順番に検索
- 最近傍法と 2-opt 改善によるブラウザ内ルート最適化
- Leaflet + OpenStreetMap タイルによる番号付きマーカーとルート線表示
- 立ち寄り件数と概算距離（緯度経度からの直線ベース）を表示
- Google マップのURLで、計算順のままナビを起動
- お迎え / お送りの切替に対応した印刷用ルート表
- 見つからない住所の行を赤く表示
- スマホ・タブレットでも使いやすいレスポンシブ表示

## 使い方

1. `index.html` をブラウザで開きます。
2. 「送迎区分」で「お迎え」または「お送り」を選びます。
3. 事業所住所を入力します。
4. 必要に応じて「最後に事業所へ戻る（周回ルート）」を切り替えます。
5. 利用者の名前と住所を行ごとに入力します。
6. 「ルートを計算する」を押します。
7. 地図、計算結果、概算距離を確認します。
8. 必要に応じて「Googleマップで開く」または「ルート表を印刷」を使います。

## GitHub Pages での公開手順

1. このリポジトリを GitHub に push します。
2. GitHub のリポジトリ画面で **Settings** を開きます。
3. **Pages** を開きます。
4. **Build and deployment** の **Source** で **Deploy from a branch** を選びます。
5. **Branch** に公開したいブランチ（通常は `main`）と `/ (root)` を指定します。
6. 保存後、表示される `https://<user>.github.io/<repo>/` のURLにアクセスします。

このアプリはCSSとJavaScriptを相対パス（`css/style.css`, `js/app.js`）で読み込むため、GitHub Pages のプロジェクトサイト配下でも動作します。

## プライバシー注意

- 入力した事業所住所・利用者名・利用者住所は、このアプリ内に保存されません。
- サーバーやデータベースは使っていません。
- 住所を緯度経度に変換する時だけ、住所文字列が OpenStreetMap Nominatim に送信されます。
- Google マップで開く時は、計算済みの住所順をGoogle マップのURLに含めてブラウザで開きます。
- 個人情報を含む住所や氏名の取り扱いは、事業所内のルールに従ってください。

## 技術構成

- HTML / CSS / JavaScript のみ
- Leaflet（CDN）
- OpenStreetMap タイル
- OpenStreetMap Nominatim API
- Google Maps Directions URL（APIキー不要）

## ローカル確認

`index.html` を直接ブラウザで開いて利用できます。ブラウザの制限や環境によってCDN・外部APIへのアクセスが制限される場合は、簡易HTTPサーバーでも確認できます。

```bash
python3 -m http.server 8000
```

その後、`http://localhost:8000/` を開きます。
